#### **5.2 Engine层**

引擎层是游戏引擎的核心业务逻辑层，构建在Foundation基础设施之上，提供完整的游戏开发能力。各模块遵循"数据依赖驱动实现"的架构原则，保持接口薄、数据契约轻与模块边界清晰。

**依赖关系（自底向上）：**
- **基础层**：Resource → Scene → Physics → Animation → Renderer → Gameplay
- **运行时层**：Runtime提供统一生命周期管理与调度框架
- **系统层**：Messaging、ModuleSystem、Input、UI提供系统服务支撑

**1. Runtime(运行时系统)[3]**
依赖：Foundation.Math、EngineCore
运行时核心系统，提供引擎生命周期管理、统一任务调度钩子和系统集成接口，负责各子系统的启动、关闭、任务调度和阶段化执行，为游戏引擎提供稳定的运行时基础。支持阶段化更新（FixedUpdate/VariableUpdate/RenderSync）和系统依赖管理。
特性：生命周期管理、任务调度、系统钩子、阶段化执行、系统集成

**2. Resource(资源系统)[3]**
依赖：Foundation.Memory、Foundation.Containers
资源管理系统，提供资产ID化、分级缓存、异步加载、依赖图和流式管线功能，支持引用计数和弱句柄管理。遵循"资产ID化 + 分级缓存 + 异步加载 + 依赖图 + Streaming管线 + 引用计数/弱句柄"架构，确保资源的高效管理和低内存占用。支持Disk Cooked Asset → CPU Cache → GPU Cache三层缓存体系。
特性：资产ID、分级缓存、异步加载、依赖图、流式处理、版本控制、句柄管理

**3. Scene(场景系统)[3]**
依赖：Foundation.Math、System
场景管理系统，提供节点层（变换/层级）、数据层（ECS）和空间层（BVH/Region/Octree）三层分离架构。节点层负责层级与变换，数据层提供ECS架构的组件系统，空间层提供场景级空间查询和加速结构，支持渲染剔除、光照剔除和区域划分。
特性：节点管理、ECS架构、空间分区、场景图、层次结构、层级变换、空间加速

**4. Animation(动画系统)[3]**
依赖：Foundation.Math、Resource、Scene
动画系统模块，提供统一骨骼、动画Graph、Blend树、曲线/约束驱动和Retarget层功能。核心架构包括统一骨骼体系、压缩曲线Clip、Animation Graph节点系统、混合系统、约束系统和Pose缓存层，支持CPU/GPU解算可切换。所有动画逻辑在Graph中，支持状态机、混合、层、分层骨骼和程序化节点。
特性：骨骼动画、动画图、混合树、曲线驱动、重定向、约束系统、CPU/GPU解算、程序化动画

**5. Physics(物理系统)[3]**
依赖：Foundation.Math、Scene
物理系统模块，提供DBVT双树、Island/Region分区和SAP/Grid等局部加速器功能，支持碰撞检测和物理模拟。物理系统采用双树架构提供高效的广相位碰撞检测，结合Island/Region分区和局部加速器，确保物理模拟的性能和准确性。模块化设计包括碰撞检测、广相位碰撞检测、射线查询和物理世界管理。
特性：碰撞检测、物理模拟、空间加速、刚体动力学、约束求解、射线查询、碰撞体管理

**6. Renderer(渲染系统)[3]**
依赖：Foundation.Math、Scene、Resource
渲染系统模块，提供RHI抽象、RenderGraph、HLSL着色器和多后端支持（Vulkan/DX12/Metal/WebGPU）。遵循RHI抽象层设计理念，提供统一的渲染接口和帧图系统，支持多图形API后端。渲染管线包括裁剪→提交→构建GPU命令→呈现，支持实时光照、阴影、材质系统和GPU驱动渲染。
特性：渲染抽象、渲染图、着色器、多后端、图形API、RHI抽象、GPU命令构建、材质系统

**7. Gameplay(游戏逻辑系统)[3]**

**8. Audio(音频系统)[3]**

音频系统模块，提供跨平台后端、低延迟混音Graph、实时空间化和流式解码功能。架构包括音频后端抽象层、低延迟混音Graph、实时空间化（HRTF/距离/声障）、流式解码与声源管理和Effects与DSP处理。支持与Scene/Physics/Resource的无缝联动，提供完整的音频体验。
