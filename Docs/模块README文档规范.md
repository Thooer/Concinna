# 模块 README 文档规范

## 1. 模块定位

* **职责**：一句话清晰描述模块的核心功能和解决的问题。
* **边界**：明确模块的责任范围，说明模块不负责的功能，避免责任蔓延。
* **外部依赖**：列出模块依赖的其他模块或外部库，包括直接依赖和间接依赖。
* **核心设计原则**：阐述模块设计的基本原则和理念，如高性能优先、接口简洁易用等。

## 2. 设计

* **模块结构**：描述模块的目录结构和主要文件的作用，如Interface/、Impl/、Sample/等目录的职责。
* **依赖关系**：使用树状图或表格形式展示模块的依赖关系，清晰说明模块与其他模块的关系。

## 3. API

详细描述模块对外暴露的所有API，包括：

### 3.1 命名空间
说明模块的命名空间结构，方便用户正确使用模块。

### 3.2 类型定义
列出模块中定义的所有类型，包括类、结构体、枚举等。

### 3.3 函数接口
对于每个函数接口，需要包含：
* **函数签名**：完整的函数声明
* **功能描述**：清晰说明函数的作用
* **参数说明**：每个参数的含义和使用方法
* **返回值**：返回值的类型和含义
* **复杂度**：算法的时间复杂度和空间复杂度（如果适用）
* **异常说明**：函数可能抛出的异常（如果适用）
* **使用示例**：简单的使用示例

### 3.4 模板支持
说明模块对模板的支持情况，包括模板参数的要求和限制。

### 3.5 范围支持
说明模块对范围-based接口的支持情况，方便用户直接操作容器。

## 4. Samples

### 4.1 测试用例
* **测试名称**：测试用例的名称
* **测试内容**：测试用例覆盖的功能和场景
* **测试结果**：测试用例的预期结果和实际结果

### 4.2 使用示例
提供详细的使用示例，包括：
* **示例代码**：完整的示例代码
* **功能说明**：示例代码实现的功能
* **执行结果**：示例代码的预期执行结果
* **注意事项**：使用过程中需要注意的问题

## 5. 最佳实践

* **命名规范**：遵循项目的命名规范，包括类名、函数名、变量名等。
* **代码风格**：保持一致的代码风格，包括缩进、换行、注释等。
* **性能考虑**：说明模块的性能特点和使用建议，如何时使用并行算法、如何选择合适的粒度等。
* **线程安全性**：说明模块的线程安全特性，如哪些接口是线程安全的，哪些不是。

## 6. 版本历史

记录模块的版本历史，包括：
* **版本号**：模块的版本号
* **发布日期**：版本发布的日期
* **主要变更**：版本的主要变更内容，包括新增功能、修复的问题等。

## 7. 文档示例

### 7.1 模块定位示例
```
* **职责**：提供基础算法和并行算法实现，为引擎其他模块提供高效的算法支持。
* **边界**：专注于算法实现，不涉及具体业务逻辑；仅依赖Language、Containers、Memory和Concurrency模块。
* **外部依赖**：
  - Language：语言基础支持
  - Containers：容器支持
  - Memory：内存管理支持
  - Concurrency：并发支持
* **核心设计原则**：
  - 高性能优先
  - 接口简洁易用
  - 支持并行执行
  - 模板化设计，支持多种类型
  - 范围-based接口，方便直接操作容器
```

### 7.2 API文档示例
```cpp
// 对迭代器范围内的元素进行排序
// 复杂度：O(n log n)
// 参数：
//   first：范围的起始迭代器
//   last：范围的结束迭代器（不包含）
// 注意：
//   - 要求迭代器支持随机访问
//   - 元素类型需要支持比较操作符<
template<typename It>
void Sort(It first, It last) noexcept;
```
